## Overview

*Last Updated - 13 March 2026*

**UniPool** is an exclusive mobile carpooling platform that prioritizes secure, cost-effective travel for university students. Now entering the **functional prototype** phase, the application solves daily commuting challenges by effectively matching students on similar routes, fostering community trust through strictly verified institutional access.

## Key Features (In Development)

### **Secure Ride System**

- **Find & Offer Rides**: Seamless interface for drivers to list empty seats and riders to find matching commutes (Map & List views).
- **Exclusive Verification**: Domain-locked signup/login (university email) with email verification + profile gating.
- **Cost Sharing**: Built-in mechanisms to equitably split commuting expenses.
- **Schedule Syncing**: Matching algorithms based on class timings and daily departure schedules.

### **Safety & Coordination Design**

- **In-app Messaging (Implemented)**: Secure, real-time 1-on-1 chat implementation for passengers and drivers.
- **Real-time Tracking (Planned)**: Live GPS updates for active carpools.
- **Custom Pickups**: Flexible coordination for convenient campus drop-off and pickup points.
- **Peer Trust Network**: Travel exclusively with verified peers from your own institution.

## Development Progress

### **Current Status**

- **Functional Prototype**: Core flows (Auth, Post Ride, Find Ride, Chat) are operational.
- **Real-time Architecture**: Firestore `onSnapshot` listeners drive the Chat, Inbox, and Activity feeds.
- **UI/UX**: Polished screens for Chat, Activity (Past/Upcoming), and Ride Details Modal.

### **Completed**

- **Auth + Verification**: Firebase email/password auth, domain restriction, email verification checks.
- **Profile Setup**: Firestore-backed profile creation + route guards (must be verified + profile complete).
- **Rides Core**: Posting rides, viewing them on Map/List, and "Requesting Seat" service logic (`arrayUnion`).
- **Messaging System**: Full real-time chat infrastructure with optimistic UI updates and an Inbox screen.
- **Ride Details Modal**: `/modal` screen loading dynamic ride data.
- **Activity Feed**: Segmented view for upcoming and past rides.
- **Design System**: Central Light/Dark theme tokens via `ThemedView`/`ThemedText`.

### **Next Up**

- **Wiring UI Actions**: Connecting the "Request to Join" and "Message" buttons in the details modal to the backend services.
- **Profile Data**: Replacing placeholder avatars with real user data from Firestore.
- **Push Notifications**: Alerting users when their ride request is accepted or a new message arrives.
